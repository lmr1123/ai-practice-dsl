# 大参林 AI 陪练可交付使用验证说明

## 1. 当前可验证范围

当前手机端网页和 Node 后端已支持一轮完整业务验证：

1. 选择场景：收银台引导办卡、益智营养类新品推荐。
2. 训前学习：查看微学习卡片。
3. 训前小测：答对 2/3 后进入对练。
4. 对练准备：单页查看顾客角色、场景说明、对话目标、优秀案例入口。
5. 固定对话模式：AI 顾客开场，员工点击说话后通过讯飞 ASR 实时转写，或手动输入真实话术。
6. 多轮追问：后端调用 GLM，根据员工话术、风险表达和当前场景，继续生成顾客追问。
7. 实时点评：输出命中动作、风险话术、润色表达、下一步灵感。
8. 练后报告：总分、是否通过、维度得分、风险提醒、改进建议、对话记录。
8. 本地记录：通过 `localStorage` 保存最近 30 条对练记录，首页可看到最近得分。

当前版本用于“正式接口链路验证”，依赖本地 Node 后端，不把密钥暴露到前端。

## 3. 验证路径

启动：

```text
npm install
npm start
```

访问：

```text
http://localhost:4173/mobile/
```

推荐验证路径：

1. 首页点击“开始对练”。
2. 查看对练准备页。
3. 点击“开始对练”。
4. 在固定对话模式页点击“说”，直接说出员工话术，观察讯飞 ASR 实时转写。
5. 再次点击“停”结束录音，确认转写内容。
6. 点击“发”，查看 GLM 生成的实时点评、润色表达和下一轮顾客追问。
7. 完成 1-3 轮后点击“查看本次报告”。
8. 返回首页，确认对练记录已更新。

## 4. 当前接口能力

- 讯飞 ASR：`WS /api/asr/stream`，前端上传 16k PCM，后端代理讯飞语音听写。
- GLM 对话：`POST /api/practice/session`、`POST /api/practice/message`、`POST /api/practice/finish`。
- 飞书记录：对练结束后通过飞书 CLI 创建或追加飞书文档；失败时落本地兜底。
- 本地记录：浏览器仍通过 `localStorage` 保存最近 30 条对练记录，便于首页展示。

## 5. 上线真实系统时需要的接口

如果进入多人试点或正式上线，需要后端提供以下接口：

### 5.1 用户与组织

```text
GET /api/me
GET /api/org/stores
GET /api/org/employees
```

用途：识别员工、门店、区域、角色权限。

### 5.2 场景与知识库

```text
GET /api/training/scenarios
GET /api/training/scenarios/{id}
GET /api/training/policies/current
```

用途：下发场景、角色卡、商品资料、会员政策、活动规则和评分规则。

### 5.3 AI 对话

```text
POST /api/practice/session
POST /api/practice/session/{id}/message
POST /api/practice/session/{id}/finish
```

用途：创建对练、生成 AI 顾客回复、保存多轮对话和结束报告。

### 5.4 语音能力

```text
POST /api/speech/asr
POST /api/speech/tts
```

用途：员工语音转文字、AI 顾客语音播报。

当前网页验证版已提供浏览器语音识别和文本兜底方案。正式生产环境建议接企业自有 ASR/TTS，且页面必须部署在 HTTPS 域名下，才能稳定获取麦克风权限。

## 5.6 前端自测

```text
npm run check
curl -s "http://localhost:4173/api/health"
curl -s "http://localhost:4173/mobile/" | rg "voiceCapture|replyInput|app.js\\?v=20260503-voice"
```

自测重点：

1. 从首页点击开始对练，进入对练准备。
2. 点击开始对练，进入 AI 顾客固定对话模式。
3. 点击语音按钮，语音可用时实时转写；不可用时自动切换文本输入。
4. 提交真实话术后，出现店员绿色气泡、实时点评、润色表达和下一轮顾客追问。
5. 点击查看报告，看到维度得分、风险提醒、改进建议和对话记录。

### 5.5 评分与报告

```text
POST /api/practice/score
GET /api/practice/reports/{id}
GET /api/practice/reports?employeeId=&storeId=&scenarioId=
```

用途：多维评分、风险话术识别、报告回看、店长看板统计。

## 6. 当前仍需业务定版的信息

正式试点前建议确认：

1. 新会员券当前政策：`3张5元=15元` 还是 `3/5/8=16元`，是否本单可用。
2. `Y-氨基丁酸多维营养素压片糖果` 的真实活动价和适用限制。
3. 第一轮员工试用名单、门店范围、通过分数和验证反馈表。
